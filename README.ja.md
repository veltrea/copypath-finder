# CopyPath

macOS の Finder の右クリック →「サービス」サブメニュー内に **「パスをコピー」** を追加する最小限のユーティリティ。

[English README](README.md)

## なぜ作るのか

macOS には Option + 右クリックでパスをコピーする隠し機能があるが、両手が必要で発見性も低い。

CopyPath はサービスメニュー経由で **「パスをコピー」** を提供する。ファイルを右クリック →「サービス」→「パスをコピー」でクリップボードに POSIX パスが入る。片手・ワンクリック。

## なぜトップレベルじゃなくサービスメニューなのか

以前のバージョンは `FIFinderSync` 拡張で右クリックメニューのトップに項目を出していたが、macOS Sequoia (15.x) で動作しなくなった。Apple の Developer Technical Support によれば、FinderSync は元々任意のファイルにコンテキストメニューを追加する用途を想定しておらず、Sequoia では右クリック統合が不安定。

サービス API が現代の macOS で唯一サポートされた方法。Hookmark や PathFinder 等の商用代替アプリも同じくサービスメニュー方式を採用している。

## 機能

- 任意のファイル・フォルダで動く（サービスメニュー経由）
- ワンクリック・修飾キー不要
- 単一選択: そのファイルの POSIX パスをコピー
- 複数選択: 改行区切りで連結してコピー
- 英語・日本語対応
- バックグラウンド常駐: 初回起動後はアプリが裏で生きるので、毎回起動せずにサービスが応答する

## インストール

```bash
./install.sh
```

ビルド → `/Applications/CopyPath.app` にインストール → LaunchServices に登録 → Finder 再起動 → サービス登録のために 1 回起動、まで一気にやる。

あるいは DMG から `CopyPath.app` を `/Applications` にドラッグして 1 回起動するだけでも OK。

## 使い方

1. Finder でファイルかフォルダを右クリック
2. **「サービス」** にホバー
3. **「パスをコピー」** をクリック

キーボードショートカット割り当て: **システム設定 → キーボード → キーボードショートカット → サービス → ファイルとフォルダ → パスをコピー**

## 必要環境

- macOS 13.0 以降
- Xcode 16 以降
- [xcodegen](https://github.com/yonsm/XcodeGen) (`brew install xcodegen`)

## 仕組み

メインアプリの `Info.plist` に `NSServices` エントリを宣言、起動時に自身をサービスプロバイダとして登録する。サービスハンドラがシステムペーストボードからファイル URL を読み、結合したパスを `NSPasteboard.general` に書き込む。

起動直後にサービス呼び出しが入ったらセットアップウィンドウを抑制するので、メニュー項目を選んでもウィンドウは出ない。

## 開発

```bash
./scripts/dev-reinstall.sh --build
```

走ってる CopyPath プロセス（前ビルドの残骸含む）を全部止め、状態を消し、再ビルド、`/Applications` に再インストール、再起動。バイナリのタイムスタンプを出力するので、ウィンドウタイトルバーと照合すれば今走ってるのが本当に最新ビルドか確認できる。
